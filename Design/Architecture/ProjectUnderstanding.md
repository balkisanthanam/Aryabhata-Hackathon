# Aryabhata — Project Understanding & Architecture Reference

## 1. Product Overview

**Aryabhata** is an AI-powered educational web app for Class 11-12 CBSE students preparing for **JEE** (Joint Entrance Examination). It digitizes NCERT textbook content (Physics, Chemistry, Mathematics) and provides:
- Structured practice by chapter/exercise
- AI-generated step-by-step solutions with progressive reveal
- AI-powered feedback on handwritten student submissions
- **(NEW) JEE Ascent**: A tiered progression system (NCERT -> Step-up -> JEE Main) with AI-curated and tagged past papers to bridge the gap between board-level and competitive formats.

**Target audience:** Class 11 & 12 CBSE students (JEE aspirants)
**Example user persona:** Visu (Class 11, CBSE, Goal: JEE)

## 4. End-to-End Request Flow

```
Student Browser (React SPA)
        ↕  HTTPS / CORS
Azure Functions v4  (Node.js/TypeScript — 11 HTTP endpoints)
        ↕
PostgreSQL (Prisma ORM)    ←→    Azure Blob Storage (images, PDFs)
                                         ↑
        Azure Queue Storage ──→ Durable Function pipeline ──→ Gemini API
                                         ↑
                      Python Extraction Pipeline (offline, NCERT PDFs)
```

Key patterns:
- React SPA calls Azure Functions API over HTTPS
- Functions query PostgreSQL via Prisma ORM
- Image assets (question figures, student uploads) stored in Azure Blob; served via SAS tokens (1-hour TTL) injected server-side
- Long-running AI evaluation jobs enqueued in Azure Storage Queue → picked up by Durable Functions pipeline

## 8. Solution Feedback Feature

The most complex feature. Lets students upload handwritten solutions and receive structured AI teacher-like feedback.

### Submission Flow
1. Student selects Subject, enters Problem Text Reference (e.g., "13.4"), uploads solution images (1–5 JPG/PNG/PDF), optionally uploads problem images
2. `POST /api/evaluations` — multipart/form-data → images uploaded to Azure Blob (`feedback/student-uploads/{userId}/{jobId}/`) → DB record created (status: PENDING) → jobId pushed to Azure Queue
3. Durable Functions pipeline processes asynchronously:
   - Read evaluation + images from blob
   - Split multi-page handwritten images per problem
   - Parse problem text reference (chapter/exercise/question number)
   - Retrieve matching NCERT question + chapter PDF from DB
   - Call Gemini vision model to evaluate against reference solution
   - Write structured `feedback_json` to DB (status: COMPLETED / FAILED)
4. Frontend polls `GET /api/evaluations/detail/{jobId}` every 10 seconds (2-minute timeout)
5. On COMPLETED: render structured feedback

### Feedback JSON Structure
```json
{
  "status": "COMPLETED",
  "ocrText": "Student's handwriting as extracted text",
  "isCorrect": false,
  "steps": [
    {
      "stepNumber": 1,
      "description": "Student wrote F = ma",
      "isCorrect": true,
      "correctionHint": null
    },
    {
      "stepNumber": 2,
      "description": "Student substituted values incorrectly",
      "isCorrect": false,
      "correctionHint": "Use $v_0 = 5$ m/s, not 0.5 m/s"
    }
  ],
  "finalTeacherNote": "Good approach overall. Watch your unit conversions."
}
```

### UI States
| State | Experience |
|-------|-----------|
| PENDING / PROCESSING | "Guru is analyzing..." with spinner; submissions disabled |
| COMPLETED | Structured feedback rendered inline |
| FAILED | Error message with retry option |

## 9. NCERT Extraction Pipeline

An offline Python pipeline that digitizes NCERT PDF textbooks into the database.

### Three Stages

**Stage 1 — Question Extraction** (`extraction_engine.py`)
- Model: `gemini-3.1-pro-preview` (pass 1, temp 0.2) — extracts questions, LaTeX formulas, exercise structure
- Model: `gemini-3-flash-preview` (pass 2) — detects and crops figures (bounding boxes on 0–1000 scale)
- Sliding window: processes pages N and N+1 together to handle text spillover
- All math in LaTeX (`$...$`), chemical formulas in mhchem (`$\ce{...}$`)
- Output: `Output/{pdf}_extraction.json`

**Stage 2 — Solution Generation** (`solver_engine.py`)
- Model: `gemini-3-pro-image-preview` (temp 0.4) — generates pedagogical step-by-step solutions
- Full PDF cached in Gemini context (TTL: 1 hour) to reduce cost/latency
- Batched 5 questions/batch; checkpointed after each batch (resumable)
- Subject-specific rules: Physics always includes "given/to find"; Chemistry uses `\ce{}` notation
- Output: `Output/{pdf}_solutions.json`

**Ingestion Phase** (within `e2e_pipeline.py`)
- Looks up `chapterid` from `(class, subject, chapter_number)`
- Uploads cropped figures to Azure Blob; replaces local paths with URLs in JSON
- UPSERTs into `exercisedata` (returns exerciseid)
- UPSERTs into `questiondata` with `content` (and `solution=null` initially)
- After Stage 2: UPDATEs `questiondata.solution` with generated solutions

### CLI Usage
```bash
# Full end-to-end
python main.py --stage 3 --pdf "Input/keph203.pdf" --class 11 --subject Physics

# Extraction only
python main.py --stage 1 --pdf "Input/keph203.pdf" --subject Physics

# Solution generation only
python main.py --stage 2 --pdf "Input/keph203.pdf" --questions all

# Cleanup
python main.py --cleanup --pdf "Input/keph203.pdf" --chapter-id 24 [--dry-run]
```

### Pipeline State (Checkpoint File)
Saved at `Output/{pdf_stem}_pipeline_state.json`. Tracks completion per stage and per question batch — allows resuming after any interruption.

## 10. Authentication

**Current status: Hardcoded single user (development only)**

```typescript
// apps/functions/src/utils/session.config.ts
export const sessionUser = { Name: 'Viswanathan', UserId: 1 };
```

On app startup, `App.tsx` calls `POST /api/auth/login` → returns hardcoded user from DB → stored in Zustand `useUserStore`. All API calls use this user context.

Real authentication (Azure AD B2C or similar) is a planned future enhancement.

## 11. Content Coverage

**Classes:** 11 and 12
**Board:** CBSE
**Subjects:** Physics, Chemistry, Mathematics

**Class 11 Chemistry (9 units):** Basic Concepts, Structure of Atom, Periodic Table, Chemical Bonding, Thermodynamics, Equilibrium, Redox Reactions, Organic Chemistry Basics, Hydrocarbons
